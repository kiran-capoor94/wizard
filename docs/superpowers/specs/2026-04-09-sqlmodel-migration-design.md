# SQLModel Migration Design

**Date:** 2026-04-09
**Status:** Approved

## Context

Wizard currently uses Peewee as its ORM. The goal is to migrate to SQLModel, which unifies SQLAlchemy (query engine, migrations) and Pydantic (validation, serialisation) in a single model definition. This eliminates the custom `EnumField`, brings native Pydantic validation to models, and enables Alembic for versioned schema migrations.

No data migration is required — the database is fresh.

---

## Architecture

Two files change, one new file, one new directory:

```
src/
  models.py     — SQLModel table classes + enums (replaces Peewee models)
  database.py   — engine, URL builder, get_session() (new)
  config.py     — settings.db default: "wizard.db" (was ":memory:")
alembic/        — migrations directory (new, generated by alembic init)
  env.py        — wired to SQLModel.metadata and app engine URL
  versions/     — generated migration scripts
alembic.ini     — Alembic config (sqlalchemy.url left blank, set in env.py)
```

All other `src/` files remain untouched. `server.py` is unchanged — DB init is Alembic's responsibility, not startup code.

---

## Models (`src/models.py`)

The custom `EnumField` is removed. SQLModel handles Python enums natively via SQLAlchemy, storing the enum **value** (e.g. `"low"`) as VARCHAR. No custom mapping needed.

Shared timestamps live in a non-table mixin. Only concrete tables get `table=True`. `MeetingTasks` is a pure join table — no timestamps, just two composite primary key foreign keys.

```python
class TimestampMixin(SQLModel):
    created_at: datetime = Field(default_factory=datetime.now, index=True)
    updated_at: datetime = Field(default_factory=datetime.now)

class TaskPriority(str, Enum): ...
class TaskCategory(str, Enum): ...
class TaskStatus(str, Enum): ...
class MeetingCategory(str, Enum): ...

class MeetingTasks(SQLModel, table=True):
    meeting_id: int = Field(foreign_key="meeting.id", primary_key=True)
    task_id: int = Field(foreign_key="task.id", primary_key=True)

class Task(TimestampMixin, table=True):
    id: int | None = Field(default=None, primary_key=True)
    name: str
    due_date: datetime | None = None
    priority: TaskPriority = TaskPriority.MEDIUM
    category: TaskCategory = TaskCategory.ISSUE
    status: TaskStatus = TaskStatus.TODO
    meetings: list["Meeting"] = Relationship(back_populates="tasks", link_model=MeetingTasks)

class Meeting(TimestampMixin, table=True):
    id: int | None = Field(default=None, primary_key=True)
    content: str
    category: MeetingCategory = MeetingCategory.STANDUP
    summary: str | None = None
    tasks: list[Task] = Relationship(back_populates="meetings", link_model=MeetingTasks)
```

**Changes from Peewee:**
- Explicit `id` primary keys (Peewee auto-created them silently)
- Enum values stored as-is (`"low"`) not names (`"LOW"`)
- `MeetingTasks` has no timestamps — pure join
- Backrefs replaced by `back_populates` on both sides of the relationship
- Enums inherit from `str` for SQLAlchemy compatibility

## Database Module (`src/database.py`)

Owns all connection concerns. Nothing else imports SQLAlchemy/SQLModel engine primitives directly.

```python
def _db_url(path: str) -> str:
    if path == ":memory:":
        return "sqlite://"
    return f"sqlite:///{path}"

engine = create_engine(
    _db_url(settings.db),
    connect_args={"check_same_thread": False},
)

@contextmanager
def get_session() -> Generator[Session, None, None]:
    with Session(engine) as session:
        yield session
```

`check_same_thread: False` is required for SQLite with a shared engine (FastMCP may dispatch across threads).

`@contextmanager` ensures the `Session` is closed immediately when the `with` block exits, not on GC. Tools and repositories consume sessions via:
```python
with get_session() as session:
    ...
```

Alembic's `env.py` imports `engine` and `_db_url` from here — single source of truth for the URL.

## Alembic (`alembic/`)

Generated via `alembic init alembic`. After init, two files are configured:

**`alembic.ini`** — `sqlalchemy.url` is left blank (URL is set dynamically in `env.py`):
```ini
sqlalchemy.url =
```

**`alembic/env.py`** — wired to the app's engine and SQLModel metadata:
```python
from sqlmodel import SQLModel
from src.database import engine, _db_url
from src.config import settings
import src.models  # registers Task, Meeting, MeetingTasks with SQLModel.metadata

config.set_main_option("sqlalchemy.url", _db_url(settings.db))
target_metadata = SQLModel.metadata

def run_migrations_online():
    with engine.connect() as connection:
        context.configure(connection=connection, target_metadata=target_metadata)
        with context.begin_transaction():
            context.run_migrations()
```

`import src.models` must appear before `target_metadata = SQLModel.metadata` — SQLModel populates metadata lazily as table classes are defined.

**Migration workflow:**
```bash
alembic revision --autogenerate -m "description"
alembic upgrade head
```

The first migration (`alembic revision --autogenerate -m "initial"`) creates `task`, `meeting`, and `meetingtasks` tables.

One field change:

db: str = Field(default="wizard.db")  # was ":memory:"

## Tests (`tests/test_models.py`)

Tests use a fresh in-memory engine per test. The `settings.db` override (`:memory:`) is applied via `monkeypatch` + module reload before importing `src.database`, using the same pattern established in `test_config.py`.

```python
@pytest.fixture(autouse=True)
def db_session(monkeypatch, tmp_path):
    import sys
    # Write a config that points the engine at an in-memory DB
    config_file = tmp_path / "config.json"
    config_file.write_text('{"db": ":memory:"}')
    monkeypatch.setenv("WIZARD_CONFIG_FILE", str(config_file))
    monkeypatch.delitem(sys.modules, "src.config", raising=False)
    monkeypatch.delitem(sys.modules, "src.database", raising=False)

    from src.database import engine
    from sqlmodel import SQLModel
    import src.models  # ensure all tables are registered
    SQLModel.metadata.create_all(engine)
    with Session(engine) as session:
        yield session
    SQLModel.metadata.drop_all(engine)
```

The six behavioral tests survive with the same assertions:
1. `created_at` timestamps are distinct across records
2. `updated_at` is set on creation
3. `due_date` is nullable
4. `meeting.tasks` relationship returns linked tasks
5. `task.meetings` relationship returns linked meetings
6. Invalid enum value is rejected (Pydantic raises `ValidationError`)
