## Approach: New modules per responsibility (Approach B)

Three new modules, each with one job. The CLI stays thin — orchestration only.

```
src/wizard/
  agent_registration.py   NEW — all agent config knowledge
  notion_discovery.py     NEW — Notion property fuzzy-matching logic
  mcp_config.py           THIN WRAPPER — delegates to agent_registration
  cli/
    main.py               EXTENDED — --agent flag, --reconfigure-notion, analytics, multi-agent uninstall
    analytics.py          NEW — query functions + terminal table formatting
```

Rejected alternatives:
- **Monolithic (A):** `mcp_config.py` would own agent detection + format generation + file I/O + registration tracking. Violates SRP. Hard to test.
- **Services layer (C):** `services.py` handles runtime concerns. Agent file manipulation and schema discovery are setup-time concerns. Mixing them makes `services.py` a god module.

## Architecture

### `agent_registration.py`

One `AgentConfig` datatype per supported agent. Five instances covering: `claude-code`, `claude-desktop`, `gemini`, `opencode`, `codex`.

Each `AgentConfig` carries:
- Config file path (resolved at call time — supports `~` expansion and `%APPDATA%` on Windows)
- Format: `json` or `toml`
- Top-level key: `mcpServers` (most agents) or `mcp` (OpenCode)
- Entry shape (standard JSON, OpenCode variant, or Codex TOML section)

Public interface — pure functions:
- `register(agent_id: str) -> None` — writes wizard MCP entry into agent config. Idempotent: overwrites wizard key, preserves all other keys. Creates config file if absent. Creates parent dirs if absent.
- `deregister(agent_id: str) -> None` — removes wizard key from agent config. No-op if file or key absent.
- `read_registered_agents() -> list[str]` — reads `~/.wizard/registered_agents.json`. Returns empty list if absent.
- `write_registered_agents(agents: list[str]) -> None` — writes `~/.wizard/registered_agents.json`.
- `scan_all_registered() -> list[str]` — fallback: checks all 5 known config paths, returns agent IDs where wizard key is present.

Error handling: malformed JSON/TOML raises `ConfigurationError` with the path. Never silently overwrites. Missing parent dirs are created.

### `notion_discovery.py`

Pure functions — no I/O beyond the Notion API call passed in:

- `fetch_db_properties(notion_client, db_id: str) -> dict[str, str]` — returns `{property_name: property_type}` for the given DB. Calls `GET /v1/databases/{db_id}`.
- `match_properties(available: dict[str, str], required_fields: list[str]) -> dict[str, str | None]` — applies three-pass matching to map Wizard field names to Notion property names:
  1. Exact name match (case-insensitive)
  2. Type match (title → `task_name`, first date → `task_due_date`, etc.)
  3. Synonym match (`"State"` → `task_status`, `"Recording"` / `"Transcript URL"` / `"Fathom URL"` / `"Krisp URL"` → `meeting_url`, etc.)
  - Returns `{wizard_field: matched_name_or_None}`

The interactive prompt (asking the user to resolve unresolved required fields) lives in `cli/main.py` only. `notion_discovery.py` is pure matching logic.

Required fields that cannot be resolved: `cli/main.py` prompts interactively. If user skips a required field (`task_name`, `task_status`, `meeting_title`), raises `ConfigurationError` — setup cannot proceed without these.

Optional fields that fail to match: silently omitted from `notion.schema`.

### `cli/analytics.py`

Query layer — takes a db session + date range, returns plain structured data:

- `query_sessions(db, start, end) -> dict` — session count, avg duration (paired session_start/session_end), tool call totals
- `query_notes(db, start, end) -> dict` — note counts by type, mental model coverage
- `query_tasks(db, start, end) -> dict` — tasks worked, avg notes/task, stale count
- `query_compounding(db, start, end) -> float` — `task_start` calls where prior notes existed ÷ all `task_start` calls

Formatting layer — separate function:
- `format_table(data: dict, start: date, end: date) -> str` — renders terminal table per spec §15 format

`wizard analytics` in `cli/main.py` calls query → format → print. `--day`, `--week`, `--from/--to` are mutually exclusive options.

### `cli/main.py` extensions

`wizard setup`:
- Adds `--agent` option (`claude-code`, `claude-desktop`, `gemini`, `opencode`, `codex`, `all`)
- Without `--agent`: interactive numbered prompt (1–6)
- After agent registration: calls `agent_registration.write_registered_agents()`
- Adds `--reconfigure-notion` flag: re-runs Notion schema discovery only, no other changes

`wizard uninstall`:
- Reads `registered_agents.json` via `agent_registration.read_registered_agents()`
- If missing: falls back to `agent_registration.scan_all_registered()`
- Calls `agent_registration.deregister()` for each registered agent

`wizard doctor`:
- Full 10-check implementation per spec §15
- `--all` flag: report all failures instead of stopping at first
- Each check is a standalone function returning `(passed: bool, message: str)`
- Check 7 (live Notion property validation) is skipped if check 2 (notion.token) failed

`wizard analytics`:
- New command
- Options: `--day`, `--week`, `--from DATE`, `--to DATE` (mutually exclusive)
- Defaults to `--week` if no option provided
- Calls `analytics.query_*` + `analytics.format_table` + prints

### `mcp_config.py`

Becomes a thin adapter. Existing `register_mcp_server()` and `deregister_mcp_server()` call `agent_registration.register("claude-code")` and `agent_registration.deregister("claude-code")` respectively. Existing callers in `cli/main.py` are unchanged at the call site.

### Bug fixes

- `tools.py` `ingest_meeting`: change `source_type` default from `"KRISP"` to `"TRANSCRIPT"`
- `pyproject.toml`: bump version from `1.1.5` to `1.1.6`

## Data Flow

### `wizard setup --agent gemini`

```
cli/main.py setup()
  1. Create ~/.wizard/ if absent
  2. Write default config.json if absent
  3. Create ~/.wizard/allowlist.txt if absent
  4. alembic upgrade head
  5. [--agent all or claude-code: copy skills]
  6. agent_registration.register("gemini")
  7. agent_registration.write_registered_agents(["gemini"])
  8. [if notion tokens set: notion_discovery flow]
  9. Print confirmation
```

### `wizard setup --reconfigure-notion`

```
cli/main.py setup()
  → notion_discovery.fetch_db_properties(notion_client, tasks_db_id)
  → notion_discovery.fetch_db_properties(notion_client, meetings_db_id)
  → notion_discovery.match_properties(available, required_fields)
  → [interactive typer.prompt for unresolved required fields]
  → config.notion.schema written to ~/.wizard/config.json
  → Print confirmation table
```

### `wizard uninstall`

```
cli/main.py uninstall()
  → agent_registration.read_registered_agents()   # or scan fallback
  → Print confirmation prompt listing all targets
  → [user confirms]
  → agent_registration.deregister(id) for each
  → rm ~/.wizard/wizard.db, config.json, allowlist.txt, skills/, registered_agents.json
  → rmdir ~/.wizard/
```

### `wizard analytics --week`

```
cli/main.py analytics()
  → resolve date range (last 7 days)
  → analytics.query_sessions(db, start, end)
  → analytics.query_notes(db, start, end)
  → analytics.query_tasks(db, start, end)
  → analytics.query_compounding(db, start, end)
  → analytics.format_table(results, start, end)
  → print
```

### `wizard doctor`

```
cli/main.py doctor()
  → run checks 1–10 in order
  → exit 1 on first failure (unless --all)
  → print pass/fail per check
```

## Error Handling

| Component | Failure mode | Handling |
|-----------|-------------|----------|
| `agent_registration` | Malformed JSON/TOML config | Raise `ConfigurationError` with path |
| `agent_registration` | Missing parent dirs | Create dirs, proceed |
| `agent_registration` | deregister on absent file | No-op (idempotent) |
| `notion_discovery` | Required field unresolvable, user skips | Raise `ConfigurationError` — setup cannot proceed |
| `notion_discovery` | Optional field unresolvable | Silently omit from schema |
| `wizard analytics` | No sessions in period | Print table with zero values |
| `wizard analytics` | DB missing | Print doctor-style message, exit 1 |
| `wizard doctor` | Check 7 (live Notion) when check 2 fails | Skip check 7 — token not available |
| `wizard doctor` | Any check fails | Exit 1 unless `--all` |

## Testing

All new tests follow existing conventions: in-function imports, `db_session` fixture, no module-level `wizard.models` imports.

| File | What it covers |
|------|---------------|
| `test_agent_registration.py` | register/deregister all 5 formats (JSON + TOML), idempotency, missing file creation, malformed file error, `registered_agents.json` read/write, scan fallback |
| `test_notion_discovery.py` | exact match, type match, synonym match, unresolved required field, optional field absent, empty DB schema |
| `test_cli_analytics.py` | query functions with seeded db fixtures, empty period, formatting output, `--day`/`--week`/`--from/--to` option parsing |
| `test_cli.py` (extended) | `--agent` flag (all 5), `--reconfigure-notion`, multi-agent uninstall, `wizard doctor` all 10 checks, source_type fix |

TDD order within each feature: red → green → refactor. Spike freely within feature branches; tests required before the implementation task is marked done.
